# Governance

Life Engine uses a phased governance model that evolves with the project's contributor base.

## Current Phase: Benevolent Dictator

While the project has a single maintainer:

- All decisions are made by the maintainer
- Decisions are documented publicly via Architecture Decision Records (ADRs) in `docs/adrs/`
- Community input is welcome via GitHub Discussions
- All decisions are public with rationale — transparency is a core value

## Phase 2: Core Team

When the project has 3 or more regular contributors:

- A core team of 3-5 people with merge rights will be formed
- Decisions are made by lazy consensus: propose, wait, merge if no objection
- Contentious decisions go to a majority vote
- The core team meets monthly via async GitHub Discussion threads
- The roadmap is a public GitHub Project board

## Phase 3: Foundation

If Life Engine reaches significant adoption:

- Consider forming or joining a nonprofit foundation (e.g. Software Freedom Conservancy)
- Elected governance body
- Formal decision-making process
- Independent of any single contributor or company

This phase is aspirational and should not be planned for prematurely.

## Communication Channels

- **GitHub Issues** — Bug reports, feature requests, plugin requests
- **GitHub Discussions** — Q&A, ideas, community conversation
- **GitHub Projects** — Public roadmap and task tracking

Real-time chat platforms (Discord, Slack, Matrix) are not used at this stage. They create communication overhead and split attention. If the community grows large enough to need real-time chat, that decision will be documented in an ADR.

## How to Participate

- Report bugs and request features via GitHub Issues
- Ask questions and share ideas in GitHub Discussions
- Submit code contributions — see [CONTRIBUTING.md](CONTRIBUTING.md) for setup and PR process
- Follow our community standards in [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md)
